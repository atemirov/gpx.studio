# CLAUDE.md — контекст проекта для агентов

## Что это за проект

Форк [gpx.studio](https://github.com/gpxstudio/gpx.studio) (MIT) — онлайн-редактора GPX-файлов.
Цели форка: (1) полная независимость от платных API (self-hosted тайлы, роутинг BRouter, открытые данные высот), (2) русский язык по умолчанию, (3) новая фича — слой публичных GPS-треков OSM (аналог `&layers=G` на openstreetmap.org).

Мастер-план — в `PLAN.md`. Карта кодовой базы — в `docs/ARCHITECTURE.md` (если файла нет — его пишет агент Фазы 1, см. PLAN.md).

## Архитектура в двух словах

Статическое SPA без бэкенда для пользовательских данных: все треки живут в браузере (IndexedDB через Dexie). Серверные зависимости — только тайлы карт, движок роутинга и данные высот.

Monorepo:
- `gpx/` — TypeScript-библиотека: модель GPX 1.1, парсинг (fast-xml-parser), сериализация, операции над треками. **Почти не трогаем** — меняем только при багфиксах, с тестами.
- `website/` — SvelteKit-приложение. Здесь вся кастомизация.
- `infra/` — наше: docker-compose для BRouter, конфиги nginx, деплой.

Стек website: SvelteKit, TypeScript, Tailwind, shadcn-svelte, MapLibre GL JS (карта), Chart.js (профиль высот), immer (state + undo/redo), Dexie (персистентность), SortableJS (дерево файлов), MDsveX (документация).

## Команды

```bash
# библиотека (собирать первой!)
cd gpx && npm install && npm run build

# сайт
cd website && npm install
npm run dev        # dev-сервер
npm run build      # production-статика
npm run test       # тесты (Vitest)
npm run lint       # ESLint + Prettier
```

Версия Node.js — см. `.nvmrc` (фиксируется в Фазе 0).

Переменные окружения (`website/.env`): все внешние сервисы конфигурируемы. Приложение обязано полностью работать с пустым `.env` (цель Фазы 2). `PUBLIC_MAPTILER_KEY` — опциональный, только для дополнительных слоёв MapTiler.

## Правила для агентов

1. **Минимальный дифф к upstream.** Мы форк и хотим подтягивать обновления оригинала. Не рефактори чужой код без необходимости. Новые фичи — отдельными модулями (пример: `website/src/lib/osm-traces/`).
2. **Одна задача — одна ветка.** Название: `feat/...`, `fix/...`, `infra/...`. В main не коммитить.
3. **Definition of done:** `npm run build` и `npm run test` зелёные; для UI — Playwright-смок; если менялась структура/команды — обновлён этот файл; коммит-сообщение на русском, описывает «что и зачем».
4. **Не выдумывать API.** Форматы внешних API (BRouter, OSM API 0.6, tile-серверы) сверять с их документацией, не по памяти. Точки интеграции перечислены в `docs/ARCHITECTURE.md`, раздел 5.
5. **Язык:** код, идентификаторы, названия веток — английский. Коммиты, документация, комментарии в PR — русский. UI-тексты — через систему i18n, никогда хардкодом.
6. **Секреты** (SSH, ключи) — только в GitHub Secrets / `.env`, никогда в коде и коммитах.
7. **Лицензия:** сохранять LICENSE и упоминание оригинального проекта в Credits. Не удалять копирайты upstream.
8. **Боевой сервер неприкосновенен без подтверждения.** Любое применение изменений на VPS (ssh, rsync, docker на сервере) — только после явного согласия человека. Конфиги готовятся локально в `infra/` и показываются вместе с планом применения.

## Внешние сервисы

| Сервис | Что | Где конфигурируется |
|---|---|---|
| OpenFreeMap | базовые векторные тайлы, без ключа | конфиг слоёв (см. ARCHITECTURE.md §4) |
| BRouter (наш VPS) | роутинг | `PUBLIC_ROUTING_URL` в .env |
| gps.tile.openstreetmap.org | raster-оверлей GPS-треков | конфиг слоёв, URL переопределяем |
| OSM API 0.6 `/trackpoints` | векторные GPS-треки (Фаза 4, ур. 2) | модуль `osm-traces` |
| Terrain/DEM | высоты точек | `PUBLIC_ELEVATION_URL` в .env |
| PMTiles в росс. S3 (страховка) | свои тайлы при проблемах с OpenFreeMap | не активировано; требования: Range + CORS, nginx-кэш на VPS |

Ограничения OSM API: bbox ≤ 0.25 кв. градуса, ≤ 5000 точек на страницу, пагинация; обязателен debounce и кэширование; уважать rate-limit (backoff при 429/509).

## Типичные грабли

- Разработка ведётся на Windows: в репо принудительно LF (`.gitattributes: * text=auto eol=lf`, `core.autocrlf=false`). НИКОГДА не перенормализуй переносы строк массово и не коммить файлы, «изменившиеся» только из-за CRLF/LF — это ломает правило минимального диффа к upstream.
- Параллельная работа — через git worktree в `..\worktrees\` (рядом с клоном, не внутри). Не создавай worktree внутри основного клона.

- Библиотеку `gpx/` нужно пересобрать после её изменения, иначе website видит старую версию.
- Слои MapLibre чувствительны к порядку: оверлей GPS-треков — под слоями пользовательских треков, над базовой картой.
- Состояние карты (позиция, слои) сериализуется в URL-hash — при добавлении слоя не сломать парсинг hash.
- MDX-документация — 56% репозитория; при глобальном grep/replace не задевать её без необходимости.
- Роутинг в dev-режиме (`localhost`/`127.0.0.1`) не работает «из коробки»: `https://graphhopper.gpx.studio/route` отдаёт `Access-Control-Allow-Origin` только для `https://gpx.studio`, для остальных origin — CORS-блок в браузере (`ERR_FAILED`, ошибка видна в консоли, не в терминале dev-сервера). Проверено вручную (curl с разными `Origin`). Это ограничение чужого бэкенда, не нашего кода — снимается только переходом на свой BRouter (задача 2.2 в PLAN.md). До тех пор роутинг локально не протестировать.
