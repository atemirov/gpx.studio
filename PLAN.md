### Инфраструктурные находки (важно для Фазы 5 и далее)

VPS (`176.123.164.99`, Ubuntu 22.04, домен — `gpx.atemirov.ru`, DNS настроен и работает) —
**не чистый сервер**, на нём уже живут личные сервисы:
- `haproxy` (Docker, `network_mode: host`) держит порт 443 как **сырой TCP-passthrough**
  (`mode tcp`, без SNI-роутинга исходно) на приватный адрес `10.10.10.1:4443` — похоже на
  VPN/мост к другому серверу пользователя (backend называется `telemt_nodes`). Это **не**
  типичный TLS-терминирующий реверс-прокси для сайтов, как предполагал изначальный план.
  Мы добавили SNI-ветку для `gpx.atemirov.ru` поверх этого (см. `infra/haproxy/haproxy.cfg`),
  не трогая существующий маршрут.
- Также на сервере: `syncthing`, `TorrServer`, и остановленный (но не удалённый, `restart: always`
  в compose) стек `n8n` + `postgres/pgvector` + `pgadmin4` в `/opt/n8n/` — **не трогать**, это не
  мусор, а чужой запаркованный проект.
- Диск: 30 ГБ всего. После чистки (journal/apt) — ~19 ГБ свободно. BRouter-сегменты ЦФО заняли
  всего 124 МБ (исходная оценка «20-40 ГБ на европейскую часть РФ» в разделе 6 — ошибочная,
  не проверенная по факту; см. `infra/brouter/README.md` — реальные размеры .rd5-тайлов
  13-36 МБ на квадрат 5×5°).
- Порт 80 свободен (используется для Let's Encrypt HTTP-01 при выпуске TLS, ещё не сделано).

## 0. Резюме и стратегическое решение

**Решение: форк gpx.studio (MIT), а не реимплементация с нуля.**

Обоснование:
1. Цель — работающий продукт («точная копия» + свои фичи), а не учебное упражнение. Форк даёт 100% паритет функциональности в день один.
2. Лицензия MIT разрешает форк, переименование, коммерческое использование. Единственное обязательство — сохранить текст лицензии и копирайт.
3. Экономия 6–10 недель агентного времени, которое перенаправляется на дифференциаторы (слой публичных GPS-треков, self-hosted инфраструктура, русификация).
4. Оригинал активно развивается — форк позволяет периодически подтягивать upstream-фиксы.

**Что получаем на выходе:**
- Работающий GPX-редактор на собственном домене и VPS.
- Полная независимость от платных западных API (тайлы, роутинг, высоты) — всё self-hosted или на бесплатных открытых сервисах.
- Фича, которой нет в оригинале: слой общедоступных GPS-треков OSM (аналог `&layers=G` на openstreetmap.org).
- Русский язык как основной.

**Оценка:** 5 фаз, ~10–15 агентных сессий, 2–4 недели календарно при контроле в режиме part-time.

## 3. Правила работы агентов

1. **Одна задача — одна ветка — один агент.** Merge в `main` только через твоё ревью диффа.
2. **Тесты — контракт приёмки.** Агент не завершает задачу, пока `npm test` и `npm run build` зелёные. Для UI-задач — Playwright-смок (страница открывается, файл грузится, трек виден).
3. **CLAUDE.md — единственный источник соглашений.** Каждый агент читает его первым. Обновление CLAUDE.md — часть definition of done, если задача меняет структуру/команды.
4. **Не переписывать — адаптировать.** Мы форк: минимальный дифф к upstream, чтобы сохранить возможность merge'ить их обновления. Крупные новые фичи — в отдельных модулях/директориях.
5. **Параллельность:** независимые задачи внутри фазы можно гнать параллельно через git worktree + tmux (у тебя есть agent-manager-skill — он для этого и подходит). Задачи с общим кодом — строго последовательно.
6. **Язык:** код и идентификаторы — английский; коммиты, комментарии к PR, документация — русский.
7. **Все агенты работают на ноутбуке**, в клоне репозитория. Infra-агент готовит конфиги локально в `infra/` и применяет на VPS через SSH только после явного подтверждения человеком. Claude Code на боевом сервере не запускается.
8. **Права в Claude Code:** без подтверждения — `npm install/run/test`, `git` внутри рабочей ветки; ручное подтверждение — `ssh`, `rsync`, `git push`, любые команды к боевому серверу.
9. **Последовательность/параллельность:** Фазы 0–1 — один агент строго последовательно (формируют фундамент). С Фазы 2 независимые задачи параллелятся: git worktree на задачу + tmux-сессия на агента (agent-manager-skill).

#### 0.Б — Промпт агенту 0.0 (бутстрап проекта)

> 1. Проверь окружение и покажи мне версии: `git --version`, `node --version`, `npm --version`, `git config --get core.longpaths`. Если Node отсутствует — остановись и скажи.
