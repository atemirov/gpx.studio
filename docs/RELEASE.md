# Чеклист релиза

Применимо и к ручному деплою (сейчас, см. `infra/deploy/README.md`), и к будущему
автоматическому (Фаза 5, не настроен — см. `PLAN.md`).

## Перед деплоем

- [ ] `cd gpx && npm run build` — без ошибок
- [ ] `cd website && npx svelte-check --tsconfig ./tsconfig.json` — не появилось новых ошибок
      (в проекте есть некоторое количество унаследованных от апстрима — см. `docs/ARCHITECTURE.md`;
      сверяться дифом, а не абсолютным числом)
- [ ] `cd website && rm -rf build .svelte-kit && npm run build` — чистая сборка без ошибок
- [ ] Если менялся `website/.env.example` — добавлена ли переменная и в реальный `website/.env`
      на машине, с которой деплоится сборка (иначе новая фича молча не заработает)

## Деплой (см. `infra/deploy/README.md` за деталями команд)

- [ ] Полная замена `www/` на VPS (`rm -rf` + свежая копия) — **не** докладывать поверх старой
      (см. граблю в `infra/deploy/README.md`: `scp -r` не чистит старые файлы, что однажды
      привело к смеси старой/новой сборки на проде)
- [ ] `docker restart gpx_nginx` после замены файлов
- [ ] Если менялась `infra/deploy/nginx.conf` или `infra/haproxy/haproxy.cfg` — синхронизировать
      на VPS **и провалидировать синтаксис до применения** (`nginx -t` в одноразовом контейнере
      с примонтированными сертификатами, см. историю коммитов инфры за примером команды)

## После деплоя — smoke-проверка на живом сайте

- [ ] `curl -sI https://gpx.atemirov.ru/` → `200 OK`, актуальный `Last-Modified`
- [ ] Открыть `/app` в браузере, **жёсткий рефреш** (Ctrl+Shift+R) — старый кэш браузера маскирует
      реальные проблемы (было несколько случаев в этой сессии, когда "баг" оказывался кэшем)
- [ ] Карта отрисовывается (не чёрный экран) — если чёрный: проверить консоль браузера на CORS-ошибки
- [ ] Загрузить тестовый GPX-файл, трек виден на карте
- [ ] Проложить маршрут инструментом роутинга хотя бы для одного профиля (bike/foot/etc.) —
      маршрут строится, не `ERR_FAILED`/CORS в консоли
- [ ] Меню слоёв: базовая карта переключается, оверлей «GPS-треки OSM» (растровый, Уровень 1)
      и слой «GPS-треки OSM» (векторный, Уровень 2, панель настроек) включаются без ошибок в консоли
- [ ] Точки интереса (Overpass): включить категорию, на зуме ≥12 появляются иконки
- [ ] Консоль браузера: нет новых ошибок сверх уже известных (сейчас — `tiles.gpx.studio/
      mapterhorn.json` CORS для 3D-рельефа, задокументировано как некритичное)

## Откат

Откат = `git revert` проблемного коммита в `main`, пересборка, повторный деплой по этому же
чеклисту. Прямого автоматического отката на VPS нет (деплой сейчас ручной, см. выше) —
это единственный способ вернуться к рабочей версии до появления настоящего CI/CD (Фаза 5).

## Известные ограничения (не баги, см. `PLAN.md` за деталями)

- 3D-рельеф в консоли иногда логирует `AJAXError: tiles.gpx.studio/mapterhorn.json` —
  не критично, `terrainSources.mapterhorn` использует другой (открытый) URL по умолчанию, эта
  ошибка — след старого кэша браузера или устаревшей сборки.
- Профиль «мотоцикл» в роутинге использует BRouter-профиль `moped` — не точный аналог, ближайший
  доступный без внешней зависимости.
- Кастомные профили BRouter (`gravel-m11n`, `ffm-long-distance`) загружены на VPS, но не
  активированы в UI — известный незакрытый баг активации через `custom_`-префикс, см. `PLAN.md`.
